# Production Current-Source Lock

Status: active source-lock evidence
Recorded: 2026-06-14
Owner: Jami Harness

## Purpose

This record locks the current official sources that affect externally influenced
production routes beyond the release-only source lock. The generated release capability
manifest uses this file to build the production acceptance matrix and to fail closed when a
route is claimed without source-lock, owner package, command, fixture, evidence, and
support-state proof.

The matrix is generated by `pnpm release:capabilities` into
`docs/generated/release-capability-manifest.json` and checked by
`pnpm release:capabilities:check`.

## Source Findings

- The official MCP specification identifies `2025-11-25` as the latest version. Harness
  support is therefore pinned to that protocol version and remains limited to the trusted
  in-process fixture adapter named in `docs/operations/mcp-source-lock.md`.
- MCP tools are server-exposed callable capabilities with names, descriptions, schemas,
  and annotations. Harness treats tool metadata as untrusted and requires policy, audit,
  trace, evidence, artifact, and redaction proof before any support claim.
- shadcn registries expect valid public flat registry JSON files. Harness may emit typed
  references to Studio UI registry outputs, but Studio UI owns registry item
  implementation, distribution, and resident rendering.
- The shadcn registry item schema records item metadata, dependencies, and files. Harness
  shared seams stay typed data contracts; they must not copy Studio UI registry
  implementation or execute UI payload code.
- Mintlify-ready local docs output is generated from source records, but `mint validate`,
  hosted Mintlify builds, and hosted public docs publishing remain unavailable until the
  exact CLI/package and hosted target are source-locked and smoke-tested.
- OpenAI Agents SDK docs describe agents as applications that plan, call tools, coordinate
  specialists, and maintain state for multi-step work. Harness-owned orchestration,
  approval, tool execution, tracing, and run-state contracts stay product grammar rather
  than direct SDK assumptions.
- OpenAI running-agents guidance covers streaming output and conversation-state strategy.
  Hosted provider support cannot be claimed until provider behavior, authentication,
  redaction, tool calls, trace, usage, retry, cancellation, and recovery fixtures exist.
- OWASP's 2025 LLM risk list includes prompt injection, sensitive information disclosure,
  supply-chain, plugin/tool, and agency risks relevant to harness policy gates.
- OWASP LLM01:2025 describes prompt injection as user prompts altering model behavior or
  output in unintended ways. Harness policy and tool metadata handling must treat external
  instructions as untrusted input.

## Generated Acceptance Matrix

`docs/generated/release-capability-manifest.json` now includes a
`productionAcceptanceMatrix` with one row per externally influenced production route. Each
row records:

- generated capability ids covered by the row where the route corresponds to a manifest
  capability claim
- planned route
- source-lock record
- owner package
- verification command and package-script gate
- evidence artifacts
- implementation paths
- fixture/evidence paths
- hosted/public claim boundary
- support state
- official source ids
- blockers and required proof for fail-closed routes

`pnpm release:capabilities:check` fails when a supported row lacks an existing source-lock
record, an owner package in the workspace inventory, a package-script gate, implementation
paths, fixture/evidence paths, evidence artifacts, or known official source ids. Fail-closed
rows must name blockers and the proof required before a public support claim.
`pnpm release:capabilities:check` also fails when any generated capability claim is not
covered by a production acceptance matrix row.

## Unsupported Or Not Claimed

- Full remote MCP transports, OAuth, resources, prompts, tasks, stdio subprocesses, and
  Streamable HTTP are not production supported.
- OpenAPI, shell, browser, code, provider-as-tool, and A2A adapters remain fail-closed
  until route-specific source locks and adapter fixtures exist.
- Hosted model provider runtime remains fail-closed; the local deterministic provider is
  the only executable provider path.
- Hosted stores, hosted observability sinks, and hosted workbench/control remain
  unavailable. Generated route output may describe missing secret/account actions, but it
  is not hosted runtime proof.
- Mintlify validation/build/publish and hosted public docs remain unavailable. Local
  generated docs and `apps/docs/docs.json` are supported local evidence only.
- Harness-side shared UI references are typed data contracts only. Studio UI owns registry
  items, token output, primitive implementation, resident rendering, and install/config
  behavior.

## Refresh Triggers

- Any official MCP version newer than `2025-11-25`.
- Any expansion from trusted MCP fixtures into stdio, Streamable HTTP, OAuth, resources,
  prompts, tasks, sampling, elicitation, or remote server behavior.
- Any route that starts using shadcn registry output beyond typed references.
- Any hosted provider adapter implementation or provider SDK/source-lock addition.
- Any OpenAPI, shell, browser, code, provider-as-tool, or A2A adapter implementation.
- Any new security claim about prompt injection, sensitive data handling, plugin/tool
  safety, agency, or supply-chain posture.
- Any public claim or hosted route that changes support state in the generated production
  acceptance matrix.
