# Support

Jami Harness support is repo-local while the project is in foundation work.

Use tracked docs and issues for reproducible bugs, missing verification evidence, package
readiness gaps, and public-claim corrections. Do not include secrets, tokens, credentials,
signed URLs, or private account material in support artifacts.

Release, npm, GitHub release, docs hosting, Vercel, Cloudflare, and other account actions
are human interventions until the release readiness gate records the authorized workflow.
