# Code Of Conduct

Jami Harness contributors should keep project work factual, evidence-backed, and
respectful.

Expected behavior:

- Keep claims tied to source files, accepted contracts, command evidence, or explicit
  human decisions.
- Protect secrets, private account material, and user data.
- Preserve the sibling boundary with Studio UI.
- Raise blockers as concrete interventions instead of hiding them behind placeholders.
- Review disagreements through the active architecture, operations, and roadmap docs.

Unacceptable behavior includes harassment, doxxing, credential exposure, knowingly false
release or security claims, and attempts to bypass project policy or verification gates.
